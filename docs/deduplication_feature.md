# 智能去重功能

## 功能说明

在播放队列视图的右上角添加了"智能去重"按钮，可以自动识别并删除重复的歌曲版本，只保留最佳版本。

## 优先级规则

算法根据以下优先级保留歌曲版本（从高到低）：

1. **原版** (100分) - 无任何版本标记
2. **Live版** (80分) - 有live标记但无伴奏
3. **特别版** (70分) - 吟唱版、纯享版、Remix等
4. **普通伴奏版** (60分) - 有伴奏但无live
5. **Live伴奏版** (40分) - 既有live又有伴奏
6. **和声伴奏版** (20分) - 有和声伴奏

**重要**：当优先级相同时，保留第一个出现的版本。

## 使用示例

### 示例1：淬炼系列
```
原始队列：
- 黄霄雲 - 淬炼 (伴奏)
- 黄霄雲 - 淬炼 (和声伴奏)
- 黄霄雲 - 淬炼

去重后：
- 黄霄雲 - 淬炼  ← 保留原版
```

### 示例2：星辰大海系列
```
原始队列：
- 黄霄雲 - 星辰大海
- 黄霄雲 - 星辰大海
- 黄霄雲 - 星辰大海 (伴奏)
- 黄霄雲 - 星辰大海

去重后：
- 黄霄雲 - 星辰大海  ← 保留原版
```

### 示例3：如歌系列（Live vs Live伴奏）
```
原始队列：
- 龚琳娜&黄霄雲 - 如歌.flac
- 龚琳娜&黄霄雲 - 如歌.flac

去重后：
- 龚琳娜&黄霄雲 - 如歌.flac  ← 保留Live版（优先级更高）
```

### 示例4：她系列（Live vs Live伴奏）
```
原始队列：
- 黄霄雲 - 她.flac
- 黄霄雲 - 她.flac

去重后：
- 黄霄雲 - 她.flac  ← 保留Live版（优先级更高）
```

## 技术实现

### 文件修改
1. **新增文件**：
   - `utils/dedup.py` - 去重算法实现
   - `tests/test_utils/test_dedup.py` - 单元测试（23个测试全部通过）

2. **修改文件**：
   - `ui/views/queue_view.py` - 添加去重按钮和功能
   - `translations/en.json` - 英文翻译
   - `translations/zh.json` - 中文翻译

### 核心算法
- **版本检测**：使用正则表达式识别live、伴奏、和声等标记
- **智能分组**：按"歌手-歌名"分组
- **优先级评分**：为每个版本计算优先级分数
- **去重保留**：每组保留最高分版本

### UI集成
- 按钮位置：播放队列视图右上角（清空按钮旁边）
- 确认对话框：去重前需要用户确认
- 结果提示：显示删除数量和保留数量

## 使用方法

1. 打开播放队列视图
2. 点击右上角"智能去重"按钮
3. 确认操作
4. 查看去重结果

## 支持的格式

算法支持中文和英文格式：
- 中文：`淬炼 (伴奏)`、`星辰大海`
- 英文：`Song (Instrumental)`、`Song Live`
- 混合：支持括号和方括号格式

## 注意事项

- 去重操作不可撤销
- 只删除重复版本，原版总是保留
- 不同歌手的相同歌曲不会被去重
- 云盘文件和本地文件可以混合去重
