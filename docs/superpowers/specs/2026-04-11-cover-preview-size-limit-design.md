# Cover Preview Size Limit Design

## Goal

收紧统一封面预览窗口的尺寸与关闭行为：

- 预览窗口整体最大不超过 `500x500`
- 图片继续在窗口内部等比缩放
- 点击大图外部区域时关闭预览

## Scope

本次只修改共享预览组件：

- `ui/dialogs/cover_preview_dialog.py`

本次只补与共享预览窗口尺寸/关闭行为相关的测试。

本次不做：

- 修改各详情页的调用方式
- 修改拖动行为
- 修改插件桥接结构
- 为预览增加缩放、滚轮、双击等额外交互

## Current State

当前共享预览窗口会先创建一个较大的对话框，再根据屏幕可用空间缩放图片。结果是：

- 整个预览窗口可能显著大于 `500x500`
- 图片外部点击关闭行为依赖现有 overlay 区域，但没有针对“窗口本身要小于等于 `500x500`”的约束测试

## Recommended Approach

采用“限制整个预览窗口最大尺寸”的方案，而不是只限制图片内容区。

原因：

- 这和需求表述“预览窗口的最大尺寸固定不超过 `500x500`”一致
- 改动只集中在共享组件，不影响调用方
- 测试边界明确，回归风险低

## Design

### Window Sizing

`CoverPreviewDialog` 的最大窗口尺寸限制为 `500x500`。

实现上保持两个原则：

- 对话框整体尺寸不超过 `500x500`
- 图片根据窗口内部可用区域等比缩放，不拉伸、不裁切

如果原图很小，窗口可以小于 `500x500`；`500x500` 是上限，不是固定值。

### Outside Click Close

“点击大图外部关闭”维持现有定义：

- 点击内容区外的窗口区域关闭
- 点击图片内容区本身不关闭，以保留拖动行为

这样不会和“按住图片区域拖动窗口”发生冲突。

### Testing

补两个方向的测试：

- 预览窗口显示后，宽高都不超过 `500`
- 点击内容区外部时仍然会关闭

现有 `Esc` 关闭、拖动、关闭时清理 loader 的测试继续保留。

## Acceptance Criteria

满足以下条件即视为完成：

1. 任意封面预览窗口的整体宽度不超过 `500`。
2. 任意封面预览窗口的整体高度不超过 `500`。
3. 图片仍按比例显示在窗口内。
4. 点击内容区外的窗口区域会关闭预览。
5. 现有 `Esc` 关闭与拖动行为不回退。
